# 对话查看器

对话查看器用于以良好排版阅读完整的 Claude Code 对话记录。

## 消息展示

查看器将对话消息按时间顺序排列展示：

- **用户消息** — 纯文本或 Markdown，右对齐气泡
- **助手消息** — Markdown 渲染，代码块语法高亮，左对齐气泡
- **工具调用** — 可折叠面板，显示工具名、输入参数、执行输出
- **思维块** — 可折叠，淡色样式，展示 Claude 的思维过程

## 会话信息

页面顶部的会话头部显示：

- 项目名称和 Git 分支
- 会话名称（点击可内联重命名）
- 会话统计：消息数、工具调用次数、token 用量
- 会话状态指示器

## 交互模式

当会话有活跃的 broker 进程时，查看器自动切换为交互模式：

- 底部出现消息输入框
- 支持发送消息、中断执行、审批权限
- 实时流式显示 Claude 的回复

!!! info "接入方式"
    查看器打开时**不会**自动连接 broker。用户需要手动点击「接入会话」或「恢复会话」。唯一例外是从工作台新建的会话，会通过 `sessionStorage` 标记自动接入。

## 启动配置

每个 broker 会话存储 `launch_config`（启动参数），在查看器中可以：

- 查看配置摘要（模型、权限模式、预算等）
- 恢复会话时预填充之前的配置

## 代码变更

查看器提供 Git 相关功能：

- 查看会话期间的 git diff
- 浏览提交历史
- 统一差异视图，支持语法高亮

## 会话 ID

查看器内部通过 `_resolveSessionId()` 选择正确的 ID：

- `session_id`（initial_id）用于 URL 和 WebSocket 连接
- `claude_session_id` 用于加载 JSONL 历史数据

当两者相同时，`claude_session_id` 为 null。
